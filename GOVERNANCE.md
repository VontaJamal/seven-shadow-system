# Governance

Seven Shadow System is governed as an open baseline standard for AI-influenced PR review quality.

## Quick Summary

- Day-to-day maintenance uses lazy consensus.
- Policy/spec contract changes require council approval.
- Compatibility guarantees follow semantic versioning boundaries.

## Governance Model

- Versioned spec with explicit compatibility policy.
- Public RFC process for behavioral and interface changes.
- Rotating maintainer council for final decisions on standard-level changes.

## Maintainer Council

- Council size: 3-5 maintainers.
- Term: 6 months, renewable.
- Rotation target: at least one seat rotates each term.

## Decision Rules

- Routine maintenance: lazy consensus.
- Spec or policy contract changes: 2/3 council approval.
- Emergency security controls: expedited majority with post-incident RFC.

## Appeals

Contributors may appeal rejected RFCs by opening an appeal issue with:

- Prior decision link.
- New evidence or risk analysis.
- Proposed compromise.

## Compatibility Contract

- Major version: breaking changes allowed with migration path.
- Minor version: backward-compatible additions only.
- Patch version: bug/security fixes with no contract break.
